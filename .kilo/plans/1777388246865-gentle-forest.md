# MCP Server Integration Plan

## Overview
Integrate Model Context Protocol (MCP) server support into the Android AI agent app, allowing users to configure MCP server URLs and authentication secrets, and enabling MCP tools/skills to be used in chat, heartbeat, and scheduled cron tasks.

## Understanding the Current Architecture

### Current Components
1. **Settings Screen** - Manages LLM provider configs (Nvidia NIM, OpenRouter, Kilo Gateway)
2. **LlmProviderClient** - Handles HTTP streaming/non-streaming calls to OpenAI-compatible APIs
3. **AgentCore** - Orchestrates chat, heartbeat, and cron jobs; assembles context from skills
4. **SettingsRepository** - DataStore-based persistence for settings
5. **Skills System** - SKILL.md files injected as context, can define tool JSON metadata
6. **Scheduled Workers** - HeartbeatWorker, CronWorker (WorkManager-based)

### Current Data Flow
- SettingsRepository → SettingsViewModel → SettingsScreen (UI)
- SettingsRepository → AgentCore (for chat/heartbeat/cron)
- AgentCore → LlmProviderClient → LLM providers
- Skills (Room DB) → AgentCore → injected as context

## MCP Integration Design

### 1. Data Model

#### New: MpcServerConfig (repositories/)
```kotlin
data class MpcServerConfig(
    val id: String,
    val name: String,
    val url: String,           // e.g., "https://my-server.com"
    val secret: String,        // Auth token/secret
    val enabled: Boolean,
    val priority: Int          // For ordering when multiple servers available
)
```

#### New: MpcToolDefinition (models/)
```kotlin
data class MpcTool(
    val name: String,
    val description: String,
    val inputSchema: Map<String, Any>,
    val serverId: String       // Link to MpcServerConfig
)
```

### 2. Repository Layer

#### Extend SettingsRepository with:
- `mpcServers: Flow<List<MpcServerConfig>>`

#### DataStore Keys to Add:
- `KEY_MPC_SERVERS` - JSON list of MpcServerConfig

### 3. Network Layer

#### New: MpcClient (MCP-specific HTTP client)
MCP uses JSON-RPC 2.0 over HTTP (or SSE for streaming). Key endpoints:
- `POST /tools/list` - List available tools
- `POST /tools/call` - Execute a tool with parameters
- `POST /resources/read` - Read MCP resources

```kotlin
@Singleton
class MpcClient @Inject constructor() {
    private val httpClient: OkHttpClient
    
    suspend fun listTools(server: MpcServerConfig): List<MpcTool>
    suspend fun callTool(server: MpcServerConfig, toolName: String, input: Map<String, Any>): ToolResult
}
```

**Authentication:** Bearer token in header (`Authorization: Bearer <secret>`)

### 4. AgentCore Enhancements

#### Context Assembly (buildMessages):
- Fetch available MCP tools from enabled servers
- Include tool descriptions in system prompt:
  ```
  Available Tools:
  - tool_name: description
    Use: When you need to [capability]
  ```
- During tool execution pause, call MCP server, then resume with result

#### New: Tool Execution Flow
1. LLM response includes tool call request (e.g., `{"tool": "search", "input": {...}}`)
2. AgentCore identifies which MCP server owns the tool
3. Execute via MpcClient.callTool()
4. Feed result back to LLM as follow-up message
5. Continue streaming response

#### For Non-Streaming (Heartbeat/Cron):
- Same tool mechanism but synchronous
- Tool results become part of the prompt context:
  ```
  Tool result: [search] Found 5 recent updates about X
  ```

### 6. Tool Discovery & Schema

**MCP Tool Manifest Format:**
Tools should be discoverable from MCP servers. The app should:
1. On server add/edit, call `listTools` endpoint
2. Cache tool definitions locally (Room DB)
3. Display in Skills screen as "Available Tools"

**Schema Structure:**
```json
{
  "tools": [
    {
      "name": "search_docs",
      "description": "Search documentation",
      "inputSchema": {
        "type": "object",
        "properties": {
          "query": {"type": "string"},
          "limit": {"type": "number"}
        },
        "required": ["query"]
      }
    }
  ]
}
```

### 7. Integration with Existing Features

#### Chat (AgentCore.chat()):
- Include MCP tool context in system prompt
- Detect LLM tool calls (e.g., "Please use search_docs tool")
- Execute tool, append result to conversation
- Continue with updated context

#### Heartbeat (AgentCore.runHeartbeat()):
- If MCP tools available, include them in context
- LLM can autonomously call tools for checks
- Example: "Use server_status tool to check API health"
- Tool results inform HEARTBEAT_OK decision

#### Cron Jobs (AgentCore.runCronJob()):
- Same as heartbeat - tools available for automated tasks
- Example: Cron job that "summarize latest commits" uses git_log tool

#### Skills System:
- Extend Skill model to include `toolDefinitions: String` (JSON)
- When skill is active, its associated MCP tools become available
- Tool definitions in SKILL.md:
  ```markdown
  ## Tools
  
  tool_name: my_custom_tool
  description: Does something useful
  mpc_server: my-server-id
  parameters:
    input: string
  ```

### 8. Data Persistence

#### Room DB Updates:
Add new DAO for tools:
```kotlin
@Dao
interface ToolDao {
    @Query("SELECT * FROM tools WHERE serverId IN (:serverIds) AND enabled = 1")
    suspend fun getToolsForServers(serverIds: List<String>): List<MpcTool>
    
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun upsert(tool: MpcTool)
    
    @Delete
    suspend fun delete(tool: MpcTool)
}
```

#### SettingsRepository:
- Cache tool list after fetching from server
- Provide `getAvailableTools()` for fast access

### 9. Network Error Handling

#### MpcClient should handle:
- Invalid/expired secrets (401) - mark server as disconnected
- Timeouts - retry with exponential backoff
- Malformed responses - safe JSON parsing with defaults
- Server unavailable - graceful degradation (skip tools)

#### User Feedback:
- Red status indicator on MpcServerCard when unreachable
- Snackbar notification: "MCP server unreachable: my-server"
- Settings screen shows last sync time per server

### 10. Security Considerations

1. **Secret Storage:**
   - Currently DataStore (encrypted at rest on Android)
   - Consider migrating to Android Keystore for secrets
   - Memory: Use `CharArray` not `String` for secrets

2. **Network Security:**
   - Enforce HTTPS (android:usesCleartextTraffic="false")
   - Certificate pinning option (advanced settings)
   - Validate SSL certificates

3. **Data Minimization:**
   - Don't log tool inputs/outputs containing PII
   - Sanitize tool calls before logging

4. **Secret Visibility:**
   - Toggle "show secret" in edit dialog (like API key)
   - Never show secret in logs or crash reports

### 12. Example MCP Use Cases

#### Use Case 1: Web Search
- Server: Brave Search MCP
- Tools: `search_web(query)`, `fetch_page(url)`
- Cron job: "Search for news every morning"
- Result: Agent summarizes and notifies

#### Use Case 2: File Operations
- Server: Local File System MCP
- Tools: `read_file(path)`, `write_file(path, content)`
- Heartbeat: "Check for log file updates"
- Agent reads file, reports anomalies

#### Use Case 3: Database Query
- Server: SQLite MCP
- Tools: `query(sql)`, `describe(table)`
- Task: "Check daily active users"
- Agent queries app DB, reports trends

#### Use Case 4: API Integration
- Server: Custom REST MCP
- Tools: `call_api(endpoint, method)`
- Task: "Fetch weather for location"
- Agent gets weather, suggests umbrella?
